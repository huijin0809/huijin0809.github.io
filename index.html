4. 그룹웨어 시스템2023-08-01 ~ 2023-09-14 Notion ERD Github

프로젝트 개요 프로젝트 회고
제목 : 그룹웨어 시스템(Model2 / 스프링 MVC 패턴) - 배포중
작업기간 : 20일
인원 : 5명

 스프링 프레임워크를 사용하여 파이널 프로젝트를 진행했습니다.
모델2 방식으로 Model, View, Controller을 분리하고, 스프링 부트 모듈과 mybatis를 사용했습니다. 매주 1회마다 현직자(위OO이텍) 멘토링이 진행되어 피드백을 받아 더욱 완성도 높은 결과물을 얻을 수 있었으며, 시연 영상, 요구사항 정의서, 테이블 정의서, ERD, WBS, 화면설계서, 업무 보고서 등 다양한 산출물을 제작했습니다.
최종적으로 프로젝트 멘토 평가에서 우수상을 수상했습니다.
산출물은 노션에서 보실 수 있습니다.
담당기능 : 회원가입/인사관리 페이지, 결재함/임시저장함 목록, 휴가신청서/매출보고서 전자결재, 연차 프로세스, 매출차트

  1. 회원가입
    • 유효성 검사
      - PK인 사원번호의 중복검사를 위해 비동기 방식인 AJAX를 이용하여 RestController를 호출합니다.
      - JQuery의 blur, test 메서드를 이용하여 비밀번호 정규식 검사와 일치불일치 검사를 수행합니다.
      - validateInputs() 메서드를 생성하여 input 태그 입력값의 공백 및 유효성 검사를 수행합니다.
    • 도로명 주소 검색 기능
      - 카카오 API를 이용한 도로명 주소 및 우편번호 검색이 가능합니다.
  2. 인사관리
    • 인사정보 페이지
      - 사원목록에서 특정 사원 클릭 시 인사정보 조회 페이지로 이동하며, 바로 수정이 가능한 form 페이지로 조회와 수정이 동시에 가능합니다.
      - 부서 테이블과 팀 테이블을 조회하여 selectbox로 출력하여 부서, 팀, 직급 등의 인사정보 수정이 가능합니다.
    • 개인정보 페이지
      - 개인정보는 조회만 가능하며, 사원의 사진과 서명 정보는 Modal을 이용하여 확인할 수 있습니다.
    • 임시 비밀번호 발급 기능
      - 비밀번호 분실시 개인정보 조회 페이지에서 관리자가 임시 비밀번호를 발급할 수 있습니다.
      - 버튼 클릭시 Modal에서 JQuery의 Math.random()을 이용하여 랜덤한 임시 비밀번호를 생성합니다.
      - 또한, 비동기 방식(AJAX)으로 생성한 임시 비밀번호로 즉시 초기화할 수 있습니다.
      - 이메일 발송 버튼 클릭시 emailJs API를 이용하여 개인정보에 등록된 사용자의 이메일로 임시 비밀번호를 발송하여 관리자와 사용자의 편의성을 높혔습니다.
  3. 연차 프로세스
    • 근속기간
      - parse()을 사용하여 사원의 입사일(String)을 LocalDate 객체로 변환합니다.
      - DateTimeFormatter.ofPattern()을 사용하여 yyyy-MM-dd 형식으로 지정합니다.
      - Period 객체의 between()로 입사일과 오늘 날짜 사이의 기간을 구하고, getYears() 또는 getMonths()로 근속년(월)수를 반환합니다.
    • 기준 연차
      - 근속기간이 1년 미만일 경우, 한달 만근시 연차가 1개씩 발생합니다.
      - 근속기간이 1년 이상일 경우, 최초 1년 만근시 15개 발생하며 이후 2년마다 발생할 기준 연차가 1개씩 증가합니다.
      - 기준 연차는 최대 25개까지만 발생 가능합니다.
    • 남은 연차 일수
      - 최근 연차 발생일로부터 오늘 날짜까지의 연차 사용 내역을 조회하고, 기준 연차에서 사용 일수만큼 차감하여 남은 연차 일수를 구합니다.
      - 연차 사용 내역을 조회하기 위해 INNER JOIN과 IN, BETWEEN 연산자로 사용한 연차 일수를 조회했습니다.
      - 집계함수 SUM과 CASE문을 이용하여 지급(P), 차감(M)에 따라 계산합니다.
  4. 전자결재
    • 휴가신청서
      - 휴가 종류(반차, 연차, 보상)를 radio 버튼으로 선택시 해당 휴가 종류의 남은 일수를 동적으로 출력하여 사용자의 편의성을 높혔습니다.
      - 선택한 휴가 종류에 따라 기간을 선택하는 input 태그를 동적으로 생성하며, selectbox 또한 남은 휴가 일수만큼만 생성됩니다.
      - 시작일과 종료일이 존재하는 연차의 경우, 선택한 휴가 일수와 휴가 시작일에 따라 휴가 종료일을 자동으로 계산하여 출력합니다.
    • 매출보고서
      - 매출보고서는 당월 기준 지난 3달을 선택할 수 있도록 selectbox를 동적으로 생성하며, AJAX로 조회하여 이미 매출보고서가 결재완료된 년월은 출력되지 않도록 했습니다.
      - +와 - 버튼으로 복수의 적요를 동적으로 추가 또는 삭제할 수 있습니다.
      - 목표액과 매출액 입력 시 자동으로 목표달성률을 계산하여 출력하여 사용자의 편의성을 높혔습니다.
      - MultipartFile로 파일 첨부 기능을 구현했으며, 파일이 첨부된 문서의 수정 페이지에서는 AJAX를 이용하여 비동기적으로 기존 파일을 삭제할 수 있습니다.
    • 결재함 / 임시저장함
      - 내 문서함의 결재함 tab에서 승인해야할 문서의 목록을 조회할 수 있습니다.
      - 기간별 조회, 문서 제목 검색, 등록일순 정렬이 가능하며 GROUP BY절을 이용하여 결재상태별 건수를 카운트하여 출력합니다.
      - 임시저장함의 문서를 checkbox로 전체선택/해제하여 일괄적으로 삭제할 수 있으며, 첨부된 파일이 있을시 파일도 일괄적으로 삭제됩니다.
      - 임시저장된 문서 클릭 시 해당 문서를 이어서 작성할 수 있는 페이지로 이동합니다.
    • 결재 프로세스
      - 결재 문서 작성 페이지에서 각각의 검색 버튼 클릭 시 Modal로 중간승인자, 최종승인자, 수신참조자를 선택할 수 있는 사원 목록 검색창을 구현했습니다.
      - Modal에서 선택한 사원의 정보를 조회하여 문서에 출력하기 위해 porm.xml에 GSON 의존성을 추가하여 기존 List를 JSON 배열로 변환합니다.
      - 문서 기안 시, Mybatis의 selectKey 구문의 order 속성을 AFTER로 지정하고 last_insert_id()를 조회하여 insert 구문을 실행한 후의 키 값을 바로 반환합니다.
      - 수신참조자 또는 복수의 적요를 동일한 key값에 작업할 시, Mybatis의 foreach 구문을 이용하여 mapper를 여러번 호출하지 않아도 되도록 구현했습니다.
      - DTO를 생성하여 현재 로그인한 사원이 해당 문서의 어떤 역할인지(role) 조회하고, role과 결재 문서의 상태에 따라 분기하여 동적인 버튼을 출력합니다.
      - 결재문서를 기안/승인/반려 시 현재 로그인한 사원의 서명이 미등록 상태일 경우 서명 등록이 가능한 프로필 수정 페이지로 이동하도록 구현했습니다.
      - draftFunction.js 파일을 생성하여 중복되는 JQuery 코드를 공통 함수로 모듈화했습니다.
  5. 메인 페이지
    • 매출차트
      - AJAX로 결재완료된 매출보고서를 조회하여 Morris.js로 도넛차트(최종 매출 현황)와 바차트(분기별 목표달성률)를 구현했습니다.
    • 봉사지도
      - 공공 데이터 포털의 OPEN API를 이용하여 회사 주변의 봉사 정보를 조회하여 지도에 마커로 표시하여 사원들의 봉사 참여를 독려합니다.

 처음으로 진행한 팀 프로젝트입니다.
기존의 VO만 분리한 모델1 방식은 단순한 구조로 개발하기 쉬웠지만, 유지보수에 어려움이 있었습니다. 이번 프로젝트부터 데이터베이스와의 상호작용을 담당하는 부분을 DAO로 모듈화함으로써, 코드의 구조화와 재사용성을 높혔습니다.
담당기능 : 상품 목록/상세 페이지, 파격할인 상품 목록 페이지, 검색 결과 페이지, 문의게시판, 마이페이지 문의글 조회

  1. 상품 목록 페이지
    • 상품 리스트
      - 상품 정보, 상품 이미지, 할인정보를 조회하기 위해 LEFT JOIN을 사용하였습니다.
      - 상품 목록 페이지 상단에는 해당(또는 전체) 카테고리의 할인 상품 상위 n개가 조회되도록 하여 직관성을 높혔습니다.
      - n개는 jsp페이지에서 원하는 값으로 변경 가능합니다.
    • 파격 할인 리스트
      - 특정 할인율 이상의 상품만 조회할 수 있는 파격 할인 목록(Discount) 메뉴를 구현했습니다.
      - 할인율이 적용된 최종 가격을 구할 때, discount 테이블의 할인 기간을 고려하기 위해 CASE문을 사용했습니다.
      - 카테고리와 정렬 선택이 가능하며, 특정 할인율 또한 jsp페이지에서 원하는 값으로 변경 가능합니다.
  2. 검색 결과 페이지
    • 검색/정렬 기능
      - search바에서 검색시 접근 가능한 상품 검색 결과 페이지 입니다.
      - 카테고리, 검색단어, 가격 범위를 선택할 수 있습니다.
      - 최신순 / 낮은가격순 / 높은가격순으로 정렬할 수 있습니다.
      - row값 변경이 가능하여 원하는 갯수만큼씩 조회할 수 있습니다.
    • 동적 쿼리 작성
      - 조회 시 기본값을 주어 기본적으로 전체 상품(기본값 공백)을 최신순(switch문의 default 사용)으로 정렬하도록 했습니다.
      - 검색시 분기가 가능하도록 parameterIndex를 이용한 동적쿼리를 작성했습니다.
  3. 상품 상세 페이지
    • 상품 정보 조회
      - 상품 목록 조회와 마찬가지로 LEFT JOIN문을 사용합니다.
      - 세션 정보를 조회하여 로그인 상태일 경우 해당 아이디의 등급에 따른 적립률을 함께 보여줍니다.
      - DecimalFormat 클래스를 이용하여 상품 가격을 원하는 형식으로 포맷화하여 출력합니다.
    • JavaScript 사용
      - 상품 단일 구매 / 장바구니 담기 경로로 분기하고, 공백검사를 하기 위해 자바스크립트를 사용했습니다.
      - 선택한 상품 수량에 따라 총 금액을 계산하여 동적으로 출력하고, 동적으로 바뀌는 상품 수량의 값을 input태그의 hidden 속성을 이용하여 넘길 수 있도록 합니다.
      - 또한, 상품 수량은 DB의 상품 재고량 만큼만 선택이 가능하도록 했습니다.
      - 해당 상품의 리뷰와 문의를 함께 조회할 수 있으며, 자바스크립트를 이용한 tab메뉴로 구현했습니다.
    • 해당 상품의 리뷰/문의
      - tab 메뉴는 각각 페이징할 수 있으며, 각각 row값 변경이 가능합니다.
      - 해당 페이지에서 문의글 작성 버튼 클릭 시 productNo값이 자동으로 적용되도록 하여 편의성을 높혔습니다.
      - 리뷰 tab에서는 리뷰 키워드(제목+내용) 검색이 가능하도록 CONCAT와 LIKE 연산자를 사용했습니다. INNER JOIN문을 사용하여 조회합니다.
  4. 문의게시판
    • 문의 게시판
      - 문의글 목록은 동적쿼리로 작성하여 문의 유형, 검색단어(작성자 또는 제목+내용) 별로 조회 가능하며 row값 변경과 페이징이 가능합니다.
      - 공개/비공개를 선택할 수 있습니다. 비공개글 접근 시 세션 정보를 조회하여 작성 아이디와 일치하지 않을 경우 비밀번호 입력 페이지로 이동하도록 분기했습니다.
      - 문의글 작성 후 작성된 문의글 상세페이지로 바로 Redirect하기 위한 키 값을 RETURN_GENERATED_KEYS를 이용하여 받아옵니다.
      - 문의글 상세보기는 답변 정보를 같이 출력하기 위해 LEFT JOIN을 사용했습니다.
    • 마이페이지 나의 문의
      - 검색 시 작성기간별 조회를 추가하여 나의 문의글 조회 시 직관성을 높혔습니다.
      - 자바스크립트를 사용하여 문의글을 체크박스로 전체 선택 / 선택해제할 수 있도록 했습니다.
      - 체크박스로 선택된 값을 getParameterValues 메서드를 이용하여 배열로 받아와 일괄적으로 삭제가 가능하도록 구현하여 나의 문의글 관리의 효율성을 높혔습니다.
      - 문의글 삭제 쿼리는 IN 연산자를 사용하여 선택한 만큼의 문의글을 삭제할 수 있는 동적쿼리로 작성했습니다.
