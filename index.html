3. 조명 쇼핑몰 2023-05-30 ~ 2023-07-05 ERD Github

프로젝트 개요 개발내용
제목 : 조명 쇼핑몰(MODEL1) - 배포중
작업기간 : 20일
인원 : 4명

처음으로 진행한 팀 프로젝트입니다.
기존의 VO만 분리한 모델1 방식은 단순한 구조로 개발하기 쉬웠지만, 유지보수에 어려움이 있었습니다. 이번 프로젝트부터 데이터베이스와의 상호작용을 담당하는 부분을 DAO로 모듈화함으로써, 코드의 구조화와 재사용성을 높혔습니다. 		- 모델1 방식을 사용 (DAO 분리)
- 쇼핑몰 내에서 관리자/회원의 페이지 분리
- 관리자/회원 레벨 및 등급 부여, 회원 탈퇴시 비활성화
- 마이페이지 내에서 배송지 관리 및 주문현황, 나의 문의/리뷰 게시글 조회 가능
- 상품 할인율 적용 및 관리, 상품의 검색 및 정렬 구현
- 장바구니 기능 구현, 일정금액 이상 구매시 무료배송 적용
- 주문 시 등급에 따른 포인트 적립, 포인트 사용 가능
- 주문(order)테이블과 상품(product)테이블의 M:N(다대다)관계를 만족시키기 위한
교차 엔티티(orderProduct) 활용
- 문의 및 리뷰 게시판 구현, 관리자 페이지에서 답변 작성 가능
담당기능
담당기능 : 상품 목록/상세 페이지, 파격할인 상품 목록 페이지, 검색 결과 페이지, 문의게시판, 마이페이지 문의글 조회 (Product / Board)

  1. 상품 목록 페이지
    • 상품 리스트
      - 상품 정보, 상품 이미지, 할인정보를 조회하기 위해 LEFT JOIN을 사용하였습니다.
      - 상품 목록 페이지 상단에는 해당(또는 전체) 카테고리의 할인 상품 상위 n개가 조회되도록 하여 직관성을 높혔습니다.
      - n개는 jsp페이지에서 원하는 값으로 변경 가능합니다.
    • 파격 할인 리스트
      - 특정 할인율 이상의 상품만 조회할 수 있는 파격 할인 목록(Discount) 메뉴를 구현했습니다.
      - 할인율이 적용된 최종 가격을 구할 때, discount 테이블의 할인 기간을 고려하기 위해 CASE문을 사용했습니다.
      - 카테고리와 정렬 선택이 가능하며, 특정 할인율 또한 jsp페이지에서 원하는 값으로 변경 가능합니다.
  2. 검색 결과 페이지
    • 검색/정렬 기능
      - search바에서 검색시 접근 가능한 상품 검색 결과 페이지 입니다.
      - 카테고리, 검색단어, 가격 범위를 선택할 수 있습니다.
      - 최신순 / 낮은가격순 / 높은가격순으로 정렬할 수 있습니다.
      - row값 변경이 가능하여 원하는 갯수만큼씩 조회할 수 있습니다.
    • 동적 쿼리 작성
      - 조회 시 기본값을 주어 기본적으로 전체 상품(기본값 공백)을 최신순(switch문의 default 사용)으로 정렬하도록 했습니다.
      - 검색시 분기가 가능하도록 parameterIndex를 이용한 동적쿼리를 작성했습니다.
  3. 상품 상세 페이지
    • 상품 정보 조회
      - 상품 목록 조회와 마찬가지로 LEFT JOIN문을 사용합니다.
      - 세션 정보를 조회하여 로그인 상태일 경우 해당 아이디의 등급에 따른 적립률을 함께 보여줍니다.
      - DecimalFormat 클래스를 이용하여 상품 가격을 원하는 형식으로 포맷화하여 출력합니다.
    • JavaScript 사용
      - 상품 단일 구매 / 장바구니 담기 경로로 분기하고, 공백검사를 하기 위해 자바스크립트를 사용했습니다.
      - 선택한 상품 수량에 따라 총 금액을 계산하여 동적으로 출력하고, 동적으로 바뀌는 상품 수량의 값을 input태그의 hidden 속성을 이용하여 넘길 수 있도록 합니다.
      - 또한, 상품 수량은 DB의 상품 재고량 만큼만 선택이 가능하도록 했습니다.
      - 해당 상품의 리뷰와 문의를 함께 조회할 수 있으며, 자바스크립트를 이용한 tab메뉴로 구현했습니다.
    • 해당 상품의 리뷰/문의
      - tab 메뉴는 각각 페이징할 수 있으며, 각각 row값 변경이 가능합니다.
      - 해당 페이지에서 문의글 작성 버튼 클릭 시 productNo값이 자동으로 적용되도록 하여 편의성을 높혔습니다.
      - 리뷰 tab에서는 리뷰 키워드(제목+내용) 검색이 가능하도록 CONCAT와 LIKE 연산자를 사용했습니다. INNER JOIN문을 사용하여 조회합니다.
  4. 문의게시판
    • 문의 게시판
      - 문의글 목록은 동적쿼리로 작성하여 문의 유형, 검색단어(작성자 또는 제목+내용) 별로 조회 가능하며 row값 변경과 페이징이 가능합니다.
      - 공개/비공개를 선택할 수 있습니다. 비공개글 접근 시 세션 정보를 조회하여 작성 아이디와 일치하지 않을 경우 비밀번호 입력 페이지로 이동하도록 분기했습니다.
      - 문의글 작성 후 작성된 문의글 상세페이지로 바로 Redirect하기 위한 키 값을 RETURN_GENERATED_KEYS를 이용하여 받아옵니다.
      - 문의글 상세보기는 답변 정보를 같이 출력하기 위해 LEFT JOIN을 사용했습니다.
    • 마이페이지 나의 문의
      - 검색 시 작성기간별 조회를 추가하여 나의 문의글 조회 시 직관성을 높혔습니다.
      - 자바스크립트를 사용하여 문의글을 체크박스로 전체 선택 / 선택해제할 수 있도록 했습니다.
      - 체크박스로 선택된 값을 getParameterValues 메서드를 이용하여 배열로 받아와 일괄적으로 삭제가 가능하도록 구현하여 나의 문의글 관리의 효율성을 높혔습니다.
      - 문의글 삭제 쿼리는 IN 연산자를 사용하여 선택한 만큼의 문의글을 삭제할 수 있는 동적쿼리로 작성했습니다.

프로젝트 개요 개발내용
제목 : 지역별 회원 게시판(MODEL1) - 배포중

1차 개인 프로젝트 후 일주일 뒤 진행한 2차 개인 프로젝트입니다.
일주일동안 댓글, 카테고리, 세션(로그인), 페이지 번호 페이징 기능 등 새로운 기능들을 추가적으로 배우고 프로젝트에 적용했습니다.
특히 세션을 이용하여 회원/비회원 또는 본인이 작성한 게시글/댓글인지에 따라 다른 view와 권한을 줄 수 있었던 점이 흥미로웠습니다. 		- 모델1 방식을 사용
- 카테고리(지역)별 게시판 구현
- 로그인, 로그아웃 기능 구현
- 회원가입 및 회원정보 수정, 회원탈퇴 구현
- 10페이지 단위의 페이징 구현
담당기능
  1. 회원
    • Session
      - 세션 기반의 로그인/로그아웃 기능을 구현하였습니다.
    • 회원가입
      - 로그아웃 상태에서만 회원가입 메뉴가 표시됩니다.
      - 중복된 아이디로는 가입할 수 없습니다.
    • 프로필
      - 로그인 상태에서만 프로필 메뉴가 표시됩니다.
      - 회원정보 수정/탈퇴 시 비밀번호가 일치해야 합니다.
  2. 카테고리
    • 카테고리 메뉴
      - 카테고리별 게시글의 수를 카운트하여 확인할 수 있도록 GROUP BY절을 사용했습니다.
      - 전체 게시글의 수도 함께 출력하기 위해 UNION ALL을 이용하여 조회했습니다.
      - 카테고리 클릭 시 해당 카테고리의 게시글만 조회할 수 있습니다.
    • 카테고리 관리
      - 로그인 상태에서만 카테고리 관리 버튼이 표시됩니다.
      - 게시글이 존재하는 카테고리는 수정/삭제할 수 없도록 버튼이 표시되지 않습니다.
      - 중복되는 이름의 카테고리는 생성할 수 없습니다.
  3. 게시판
    • 게시글/댓글 관리
      - 로그인 상태에서만 글쓰기 버튼과 댓글 입력란이 표시됩니다.
      - 로그인 아이디와 작성자 아이디가 일치하는 경우에만 게시글/댓글의 수정/삭제 버튼이 표시됩니다.
    • 페이지 번호 페이징
      - 10페이지 단위의 페이지바를 만들어 더욱 직관적인 페이징을 구현했습니다.
      - 카테고리 클릭 후 페이징시에도 카테고리 선택이 유지됩니다.
      - 댓글 목록도 페이징이 적용됩니다.

VO만 분리한 모델1 방식의 간단한 기능이지만 개발에 흥미를 가지고 의욕적으로 참여할 수 있는 계기가 되었고, 기초적인 개념과 프로젝트 개발에 대한 자신감이 쌓이는 좋은 기회가 되었습니다. 		- 모델1 방식을 사용
- 게시글 리스트와 일정 리스트로 분리
- 일정은 달력 형태로 확인 가능한 다이어리로 구현
- 게시글은 간단한 게시판 형태로 확인 가능
담당기능
  1. 달력
    • Calendar API 사용
      - JAVA의 기본 라이브러리 중 하나인 Calendar API를 사용하여 달력을 구현했습니다.
    • 일정 미리보기
      - 달력의 해당 날짜의 일정을 작성시 선택한 색상과 함께 미리 보여줍니다. (5글자 제한)
      - 날짜 클릭 시 해당 날짜의 일정 목록과 일정 추가가 가능한 페이지로 이동합니다.
  2. 다이어리/게시판
    • 시작 페이지
      - 홈에서 최근 게시글 목록을 확인할 수 있습니다.
      - 홈에서 오늘 날짜의 일정 목록을 확인할 수 있습니다.
    • 일정/게시글 관리
      - 일정 메모는 색상 선택이 가능하며, 조회 시 해당 색상으로 출력됩니다.
      - 게시판은 이전/다음 버튼을 이용한 간단한 페이징이 가능합니다.
      - DML을 이용하여 일정/게시글 CRUD가 가능하도록 했습니다.
      - 일정/게시글을 수정 또는 삭제하려면 작성시 입력한 비밀번호가 일치해야 합니다.
  3. CSS
    • Bootstrap5
      - Bootstrap5를 이용하여 간단한 CSS 작업을 했습니다.
